# Smart Gateway Configuration Manual

## Working with Logs

The Logs section allows you to view and analyze all the activity logs generated by your SAP Links. Logs come from three sources: **Producer**, **Consumer**, and **SAP**.

---

## Step 1: Access the Logs Section

1. Open your SAP Links dashboard.
2. Click on the **"Go to Logs"** button.
3. A table will appear showing the most recent log entries.

---

## Step 2: Filter the Logs

Use the controls at the top of the screen to narrow down the logs you want to see:

| Field | Description |
|---|---|
| **Search** | Type any keyword to search within log messages. |
| **Last / Range** | Toggle between `Last` (relative time) or `Range` (specific dates). |
| **Time** *(Last mode)* | Select a relative time window: `Last Minute`, `Last 5 Minutes`, `Last Hour`, `Last 6 Hours`, `Last 12 Hours`, `Last 24 Hours`, `Last 7 Days`. |
| **Start Date** *(Range mode)* | Pick a start date for the log range. |
| **Start Time** *(Range mode)* | Pick a start time in 12-hour AM/PM format. |
| **End Date** *(Range mode)* | Pick an end date for the log range. |
| **End Time** *(Range mode)* | Pick an end time in 12-hour AM/PM format. |
| **Level** | Filter by severity: `ALL`, `ERROR`, `WARN`, or `FATAL`. |
| **Source** | Filter by origin: `ALL`, `PRODUCER`, `CONSUMER`, or `SAP`. |

> ⚠️ **Important — Time Zone:** All dates and times are handled in **UTC** automatically. You only need to select the time on your local clock — the system converts it for you. No manual conversion is required.

---

## Step 3: Understanding the Log Table

Each row in the table represents one log entry with the following columns:

| Column | Description |
|---|---|
| **Time** | When the log occurred (e.g. `Jul 1, 9:14 AM`). |
| **Level** | A colored badge indicating severity: red for `ERROR`, orange for `WARN`, dark red for `FATAL`. |
| **Source** | Where the log came from: `PRODUCER`, `CONSUMER`, or `SAP`. |
| **Logger** | The name of the component that generated the log. |
| **Message** | A preview of the log message (truncated to 150 characters). |
| **Actions** | A button to view the full log message in a pop-up window. |

---

## Step 4: Viewing the Full Log Message

If you need to see the complete log entry:

1. Click the document icon 📄 in the **Actions** column.
2. A pop-up window will open showing the full details:
   - Complete timestamp with seconds.
   - Level, Source, and Logger information.
   - The full message text in a scrollable area.
3. Click **"Copy Message"** to copy the entire message to your clipboard.
4. Click **"Close"** to dismiss the pop-up.

---

## Step 5: Exporting Logs to Excel

To download the logs as an Excel file:

1. Click the download icon 📥 in the filter bar.
2. The file will be saved as `ONE_CONNECT_LOGS_YYYY-MM-DD.xlsx`.
3. It contains up to 1000 log entries with the columns: Time, Level, Source, Logger, and Message.

---

## Step 6: Setting Up Email Notifications (Optional)

You can configure email alerts to be notified when `ERROR` or `FATAL` logs occur.

### Enabling the Feature

The **"Add Email"** button visibility is controlled by the `NEXT_PUBLIC_SMTP_ENABLED` environment variable. When set to `"true"` (default), features that require an SMTP mail server are enabled, including email notifications for logs.

| Setting |
|---|
| `NEXT_PUBLIC_SMTP_ENABLED=true` |

> 📌 **Note:** This is a system-level configuration. If the SMTP server is not available, your administrator can disable this setting by setting `NEXT_PUBLIC_SMTP_ENABLED=false`. For a complete reference of environment variables, see the [Environment Variables Reference](./09-Environment_Variables.md).

### Managing Email Recipients

Once the button is visible:

1. Click the **"Add Email"** button located next to the page title.
2. Add the email addresses that should receive notifications.
3. Notifications are sent when `ERROR` or `FATAL` logs are detected.

> ℹ️ To prevent excessive emails, the system only sends one notification per error type every 60 minutes.

---

## Play / Pause Function

| Mode | Behavior |
|---|---|
| **Play** ▶️ *(default)* | Automatically refreshes the log table every 10 seconds. |
| **Pause** ⏸️ | Stops the automatic refresh so you can review logs without interruptions. |

Click the Play/Pause button to toggle between modes.

---

## Pagination

At the bottom of the table, you can:

- Select how many rows to display: **10**, **15**, or **25**.
- Navigate between pages using the arrow buttons ◀ ▶.
- View the current page and total pages (e.g. `1 - 5`).

---

## Summary

The complete workflow for working with logs is:

1. Access the Logs section from your SAP Links dashboard.
2. Apply filters (Search, Time Range, Level, Source) to narrow down results.
3. Review the log entries in the table.
4. Click the document icon to view a full log message.
5. Export logs to Excel if needed for offline analysis.
6. Optionally, enable email notifications for `ERROR` and `FATAL` logs.
